这里整理了公网演示、前端操作、骨骼标准资产、Unreal 实时联动和数据导出的最短路径。公开仓库只包含静态网站和可再分发测试资产。

前端 Demo 使用

页面默认中文，可切换英文。左侧输入创意描述，点击“一键生成预演”即可同时生成动作、运镜和灯光。中间是 3D 预览舞台，右侧展示关键帧和联动信息。

  • 默认预览：加载 debug_native_glb.glb，用于 Kimodo 兼容骨骼测试。
  • 参考视觉：使用 REPO.png 作为原型角色外观参考。
  • 下载资产：提供 GLB 和 USDZ 两种骨骼标准资产，用于第三方工具测试。

骨骼标准与资产

Kimodo GLB

assets/models/debug_native_glb.glb 是默认测试角色，用于验证 Kimodo 骨骼和浏览器动作预览。

Hunyuan USDZ

assets/models/IozsL7ZmHiVHopD-D3_ay.usdz 是混元 AI 兼容的原型资产，保留用于与腾讯混元 3D 流程对齐。

REPO.png 作为视觉风格参考。旧的 FBX 轻量原型已不再作为驱动标准推荐，以避免用户下载到不符合骨骼链路的资产。

GitHub Pages 发布

  1. 提交并推送到 EndToFx/StageRehearsal 的 main 分支。
  2. 打开 GitHub 仓库的 Settings -> Pages。
  3. Source 选择 Deploy from a branch。
  4. Branch 选择 main，Folder 选择 / (root)。

Unreal 实时联动

公网 Demo 不能直接访问本机 Unreal。需要在本机运行 WebSocket bridge，然后让页面连接 ws://localhost:9090。

python bridge/ue_bridge.py --port 9090

建议推送 FPS 保持在 8-10，避免编辑器和浏览器过载。

数据导出与第三方导入

点击“导出 JSON”会下载包含 animation、camera、lighting 的结构化文件。Unreal、Blender、TouchDesigner 都可以通过脚本读取该 JSON，并映射到相机、灯光和角色控制器。

{
  "animation": [],
  "camera": [],
  "lighting": []
}
